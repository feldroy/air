# Air Reference

Here is the Air reference documentation. It explains how to do things, as well as providing reference details for nearly every object in the Air code.

- [Applications](https://docs.airwebframework.org/api/applications/index.md) - The app instantiator for Air
- [Background Tasks](https://docs.airwebframework.org/api/background/index.md) - Background tasks for Air
- [Exception Handlers](https://docs.airwebframework.org/api/exception_handlers/index.md) - Exceptions are returned to the user, specifically 404 and 500
- [Exceptions](https://docs.airwebframework.org/api/exceptions/index.md) - Sometimes it's good to know exactly what is breaking
- [Forms](https://docs.airwebframework.org/api/forms/index.md) - Receive and validate data from users on web pages
- [Layouts](https://docs.airwebframework.org/api/layouts/index.md) - Utilities for building layout functions and two example layouts for css microframeworks (mvcss and picocss)
- [Middleware](https://docs.airwebframework.org/api/middleware/index.md) - Middleware for Air
- [Requests](https://docs.airwebframework.org/api/requests/index.md) - HTMX utility function that can be used with dependency injection
- [Responses](https://docs.airwebframework.org/api/responses/index.md) - AirResponse for normal responses and SSEResponse for Server Sent Events
- [Routing](https://docs.airwebframework.org/api/routing/index.md) - For compositing multiple apps inside each other
- [SVG](https://docs.airwebframework.org/api/svg/index.md) - Reference for the entire SVG tags specification, all of which are supported by Air
- [Tags](https://docs.airwebframework.org/api/tags/index.md) - Reference for the entire HTML tags specification, all of which are supported by Air
- [Templating](https://docs.airwebframework.org/api/templating/index.md) - Describes Jinja and Air Tag renderers for use in both projects and third-party installable packages
- [Utils](https://docs.airwebframework.org/api/utils/index.md) - Utilities that don't fall into one of the above categories
